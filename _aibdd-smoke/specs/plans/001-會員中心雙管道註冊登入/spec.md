## 1. 產品目標

擴充會員中心註冊與登入方式，由目前僅支援 Email 擴展為 **Email 或台灣手機號碼** 二擇一，透過**統一輸入欄位**自動識別。

- **台灣用戶**偏好以手機號碼快速註冊（目前僅支援台灣門號 +886）
- **外國客人**或無台灣門號者可使用 Email
- 註冊後可**綁定另一種聯絡方式**，提升帳號安全性與使用便利性

### 1.1 範圍

- **包含**：雙管道註冊、雙管道登入（密碼 + OTP 驗證碼登入）、App 端生物辨識登入（Face ID / Touch ID / 指紋）、帳號綁定（補綁 Email / 手機）、編輯已綁定聯絡方式、解除綁定、忘記密碼（手機走 SMS OTP）、聯絡身分檢查擴充、防重複帳號機制、刪除帳戶
- **不包含**：第三方社群登入（LINE、Google 等）、Web 端生物辨識登入、CRM 後台手動建立會員流程

## 2. 使用者故事

- **US-01**（手機註冊）：身為一名台灣新客人，我希望用手機號碼和密碼就能快速註冊會員中心，不需要填寫 Email，讓我更快完成註冊。
- **US-02**（Email 註冊）：身為一名外國旅客，我希望用 Email 和密碼就能註冊會員中心，因為我沒有當地手機號碼。
- **US-03**（註冊後補綁）：身為已註冊會員，我希望能在帳號設定中補綁 Email 或手機號碼，讓我可以用兩種方式登入，也增加帳號安全性。
- **US-04**（手機登入）：身為已註冊會員，我希望用手機號碼和密碼就能登入會員中心，不需要記住 Email。
- **US-05**（手機忘記密碼）：身為以手機號碼註冊的會員，當我忘記密碼時，希望透過手機簡訊 OTP 驗證身份後重設密碼。
- **US-06**（首次綁定）：身為 CRM 已有資料的客人，當我用手機號碼或 Email 註冊時，如果系統發現我的資料已存在，應該被導向帳號綁定流程而非建立重複帳號。
- **US-07**（刪除帳戶）：身為已註冊的會員中心使用者，希望能刪除我的會員中心帳戶。
- **US-08**（編輯已綁定聯絡方式）：身為已註冊會員，希望能修改我已綁定的 Email 或手機號碼。
- **US-09**（解除綁定）：身為已綁定 Email 和手機的會員，希望能解除綁定其中一種聯絡方式。
- **US-10**（OTP 驗證碼登入）：身為已註冊會員，希望能選擇用驗證碼登入，而不需要輸入密碼（Email 收驗證碼、手機收簡訊驗證碼）。
- **US-11**（生物辨識登入 App 端）：身為 App 端的已註冊會員，希望能用 Face ID / 指紋等生物辨識快速登入。

## 3. 業務規則

### 3.1 註冊規則
- **BR-01** 註冊方式二擇一：選擇 Email 或手機號碼其中一種，另一種為選填。
- **BR-02** Email 格式驗證。
- **BR-03** 手機號碼格式：僅限台灣 +886；輸入 `09xxxxxxxx` 或 `9xxxxxxxx`，前端正規化後送 `+886` + 9 碼。
- **BR-04** 密碼規則：最少 8 字元。
- **BR-05** Email 不做 OTP 驗證。
- **BR-06** 手機號碼需 SMS OTP 驗證。
- **BR-07** 唯一性約束：同 Email 或同手機號碼不可重複註冊。

### 3.2 登入規則
- **BR-08** 統一欄位兩階段處理：第一階段識別 → 第二階段驗證；含 `@` → Email，純數字（可帶前導 0）→ 手機，其餘 → 格式無法識別。
- **BR-09** 手機固定 +886，前端自動帶入，無國碼選擇器。
- **BR-10** 登入節流：沿用現有機制。
- **BR-51** OTP 登入支援雙管道：Email/手機皆支援；手機走 SMS OTP，Email 走 Email OTP。
- **BR-52** OTP 登入用途不互通：LOGIN 用途 OTP 與 REGISTER / FORGOT_PASSWORD OTP 不互通。
- **BR-53** OTP 驗證成功直接完成登入。
- **BR-54** 可在 OTP 與密碼模式來回切換。
- **BR-55** OTP 登入保留 Captcha。
- **BR-56** Email OTP 透過現有郵件服務發送；有效期 / 重送 / 每日上限與 SMS OTP 一致。
- **BR-57** 生物辨識僅限 App 端。
- **BR-58** 生物辨識需先以密碼/OTP 登入後啟用，啟用時通過一次生物辨識驗證。
- **BR-59** App 使用裝置原生 API（iOS Keychain；Android Keystore），refresh token 加密儲存於裝置，不上傳雲端。
- **BR-60** App 啟動時若已啟用 → 自動提示驗證 → 通過後使用儲存 token 完成登入。
- **BR-61** 生物辨識失敗或取消 → 退回登入頁。
- **BR-62** 使用者可隨時停用；更換密碼後所有裝置 token 自動失效。

### 3.3 帳號綁定規則
- **BR-11** 綁定方向：Email 註冊→可補綁手機；手機註冊→可補綁 Email。
- **BR-12** 綁定唯一性：綁定的聯絡方式不可已被其他帳號使用。
- **BR-13** 綁定手機需 SMS OTP 驗證。
- **BR-14** 綁定 Email 不需 OTP。
- **BR-15** 綁定後兩種方式皆可登入。

### 3.4 防重複帳號規則
- **BR-16** CRM 已有 Customer 且已註冊會員中心 → 拒絕註冊，提示「此帳號已存在，請直接登入」。
- **BR-17** CRM 已有 Customer 但未註冊會員中心 → 導向帳號綁定，將既有 Customer/Member 與會員中心帳號綁定。
- **BR-18** CRM 同時有 Email 和手機的 Customer 紀錄 → 同一人則導向綁定；不同人則資料衝突需人工處理。
- **BR-19** 同一 Customer 不可有兩個會員中心帳號。

### 3.5 忘記密碼規則
- **BR-20** Email 忘記密碼沿用現有流程（直接重設，不需 OTP）。
- **BR-21** 手機忘記密碼需 SMS OTP 驗證。
- **BR-22** OTP 有效期限可由系統參數控制（建議 5 分鐘）。
- **BR-23** OTP 重送間隔至少 60 秒；每日同一號碼有發送上限。
- **BR-24** OTP 長度與格式由後端定義。

### 3.6 向下相容規則
- **BR-25** 既有 Email 會員不受影響。
- **BR-26** 既有會員可至帳號設定補綁手機。

### 3.7 前端正規化與格式驗證規則
- **BR-27** 手機號碼前端正規化：去除前導 0、補 `+886`，最終格式 `+886` + 9 位數字（呼叫後端前必先正規化，含 check-contact-identity）。
- **BR-28** 統一欄位無法識別格式 → 提示「請輸入有效的 Email 或手機號碼」。
- **BR-29** 台灣手機格式：`+886` + 9 開頭的 9 位數字。

### 3.8 前端行為規則
- **BR-30** 統一欄位於 blur 時觸發識別。
- **BR-31** Captcha 沿用，註冊/登入/忘記密碼均保留。
- **BR-32** 「記住我」支援手機，記住使用者原始輸入格式（如 `0912345678`）。
- **BR-33** 後端回傳錯誤訊息由前端直接顯示，不做轉換/i18n。
- **BR-34** 登入維持兩階段流程：第一階段帳號 + check-contact-identity；第二階段密碼。
- **BR-35** 註冊不使用 check-contact-identity；直接提交至後端。
- **BR-36** OTP 驗證錯誤次數、SMS 發送頻率（含 IP 維度）由 CRM Java 後端處理；前端僅顯示後端 error code / error msg。
- **BR-37** OTP 簡訊透過 IT 服務簡訊架構發送（寫入 SMS_SEND_INFO_LIST，由 bacchusTask 排程；簡訊廠商 twsms.com）。

### 3.9 刪除帳戶規則
- **BR-38** 刪除帳戶需密碼確認；密碼錯誤則拒絕。
- **BR-39** 軟刪除 MemberAuth，不可恢復。
- **BR-40** CRM Customer / 消費紀錄 / 積分 / 優惠券保留不刪。
- **BR-41** 刪除成功後清除前端 token，導向登入頁/首頁。
- **BR-42** 原綁定聯絡方式可重新註冊；check-contact-identity 應回傳 NEW。
- **BR-43** 刪除帳戶需二次確認（警告提示）。

### 3.10 編輯與解除綁定規則
- **BR-44** 編輯已綁定 Email：新 Email，不需 OTP。
- **BR-45** 編輯已綁定手機：新手機 + SMS OTP 驗證。
- **BR-46** 編輯唯一性檢查：新聯絡方式不可已被其他帳號使用。
- **BR-47** 解除綁定前提：Email 和手機皆已綁定才可解除其中一種。
- **BR-48** 解除綁定不需密碼確認，僅二次確認彈窗（v1.7 變更）。
- **BR-49** 解除綁定後該聯絡方式不可再用於登入；另一種仍可登入。
- **BR-50** 解除綁定後該聯絡方式釋放，可被其他使用者註冊或綁定。

## 4. 聯絡身分檢查流程（check-contact-identity）

登入前預檢查 API。**註冊不呼叫此 API**（BR-35）。

| 結果      | 說明                      | 前端導向                  |
| ------- | ----------------------- | --------------------- |
| `NEW`   | CRM 無此聯絡方式紀錄            | 註冊頁面（v1.2 後僅登入用） |
| `BIND`  | CRM 有 Customer 但未註冊會員中心 | 帳號綁定頁（首次綁定） |
| `LOGIN` | 已有會員中心帳號                | 登入頁面 |

呼叫前端正規化：手機號碼必須先 BR-27 正規化。

## 5. 核心流程

### 5.1 手機號碼註冊流程
1. 統一欄位輸入手機 + 密碼。
2. 前端 blur 識別手機 → 正規化 → 顯示 OTP 區塊。
3. 點擊「發送驗證碼」→ 呼叫 sign-up/mobile API（含 Captcha）。
4. 後端發 SMS OTP，回 OTP uuid。
5. 輸入 OTP → verify-otp。
6. 驗證成功 → 建立 Customer + Member + MemberAuth + Profile。
7. 失敗 → 回錯誤訊息，前端直接顯示。

### 5.2 Email 註冊流程（既有，不變）
1. 統一欄位輸入 Email + 密碼。
2. 前端 blur 識別 Email。
3. 呼叫 sign-up/mail API（含 Captcha）。
4. 後端建立 Customer + Member + MemberAuth + Profile。
5. 成功或帳號已存在錯誤訊息。

### 5.3 帳號綁定流程（首次註冊時觸發）
1. 輸入聯絡方式。
2. check-contact-identity → 回 BIND。
3. 提示綁定。
4. 設定密碼（手機綁定需先 OTP）。
5. 建立 MemberAuth + MemberCenterProfile（沿用既有 Customer + Member）。
6. 綁定成功，導向登入。

### 5.4 手機忘記密碼流程
1. 統一欄位輸入手機。
2. blur 識別 → 正規化 → 顯示 OTP 區塊。
3. 發送驗證碼 → forgot/mobile API（含 Captcha）。
4. 後端發 SMS OTP，回 OTP uuid。
5. 輸入 OTP → verify-otp。
6. 進入重設密碼畫面 → forgot/reset API。
7. 後端更新密碼 → 導向登入。

### 5.5 補綁流程（已登入狀態）
1. 帳號設定 → 點擊「綁定 Email/手機」。
2. 輸入聯絡方式（手機需 OTP 驗證）。
3. 後端唯一性檢查 → 更新 Customer.email / mobilePhone。
4. 完成後兩種方式皆可登入。

### 5.6 刪除帳戶流程
1. 帳號設定 → 「刪除帳戶」。
2. 顯示警告：「刪除後無法恢復」。
3. 「確認刪除」→ 顯示密碼欄位。
4. 輸入密碼 → 提交。
5. 後端驗密 → 軟刪 MemberAuth。
6. 清除前端 token → 導向登入頁。

### 5.7 編輯已綁定聯絡方式流程
1. 帳號設定 → 「編輯」。
2. 編輯手機：OTP 驗證；編輯 Email：直接輸入。
3. 後端唯一性檢查（BR-46）。
4. 更新 Customer.email / mobilePhone。
5. 頁面顯示新值。

### 5.8 解除綁定流程
1. 帳號設定 → 「解除綁定」。
2. 僅一種綁定 → disabled / 隱藏（BR-47）。
3. 顯示確認彈窗：「此 Email / 手機號碼將無法用於登入」。
4. 確認 → 後端移除該綁定（BR-48 不需密碼）。
5. 頁面更新顯示「綁定」。

### 5.9 OTP 驗證碼登入流程
1. 統一欄位輸入 Email / 手機（第一階段）。
2. 正規化（手機）→ check-contact-identity → 回 LOGIN。
3. 第二階段：遮罩帳號 + 密碼 + Captcha + 「改用驗證碼登入」連結。
4. 點擊「改用驗證碼登入」（BR-51）。
5. 密碼欄位隱藏，顯示 OTP 區塊 + Captcha（BR-55）。
6. 完成 Captcha → 發送驗證碼。
7. 後端依帳號類型發 OTP：手機走 SMS、Email 走 Email OTP；回 OTP uuid。
8. 輸入 OTP → 驗證。
9. 成功直接登入（BR-53）；失敗顯示錯誤，可重輸/重發。
10. 可隨時點「改用密碼登入」切回（BR-54）。

### 5.10 啟用生物辨識登入流程（App 端）
1. 已登入 → 帳號設定 → 「啟用生物辨識」。
2. App 呼叫裝置原生生物辨識 API（驗證本人）。
3. 通過 → App 向後端請 refresh token。
4. 後端核發 token → App 加密儲存於裝置安全區域（BR-59）。
5. 啟用完成。
6. 驗證失敗/取消 → 不啟用。

### 5.11 生物辨識快速登入流程（App 端）
1. 開啟 App。
2. 檢查：是否已啟用生物辨識（BR-58）且裝置支援。
3. 是 → 自動顯示生物辨識提示（BR-60）。
4. 通過 → App 用儲存 token 向後端驗證。
5. Token 有效 → 直接登入。
6. Token 失效（如改密碼，BR-62）→ 清除 token → 導向登入頁，提示「請重新登入」。
7. 生物辨識失敗 / 取消 → 顯示登入頁面（BR-61）。

## 6. 前端頁面異動（摘要）

- **6.1 註冊頁面**：統一輸入框 + 密碼 + Captcha；blur 識別；手機顯示 OTP 區塊；移除 Tab 切換；不呼叫 check-contact-identity（BR-35）。
- **6.2 登入頁面**：兩階段（BR-34）；統一輸入框；blur 識別；手機正規化；記住我支援手機（BR-32）；「改用驗證碼登入」連結（BR-51）；可切回密碼（BR-54）。
- **6.3 忘記密碼頁面**：統一輸入框 + Captcha；移除國碼選擇器；blur 識別決定 Email 重設 vs SMS OTP。
- **6.4 帳號設定頁面**：顯示綁定狀態；未綁定顯示「綁定」；已綁定顯示「編輯」；雙綁定時顯示「解除綁定」（BR-47）；底部「刪除帳戶」紅色警告按鈕。
- **6.5 App 端登入與生物辨識**：App 開啟自動偵測；通過直接登入；失敗回登入頁；帳號設定有「啟用/停用」開關（BR-62）。

| Key | 建議預設值 | 說明 |

| `wrs_otp_daily_limit` | 10 | 同號碼每日 OTP 發送上限 |

## 8. 依賴項目

- IT 服務簡訊架構（SMS_SEND_INFO_LIST + bacchusTask + twsms.com）
- Email 發送服務（OTP 登入用）
- 裝置原生生物辨識 API（iOS LocalAuthentication；Android BiometricPrompt）
- 裝置安全儲存（iOS Keychain / Android Keystore）
- 手機號碼 hash 機制（類似既有 Email hash）
- 手機號碼唯一索引（mobile_phone_hash unique）

## 9. 風險與注意事項

- SMS 成本（OTP 頻率限制緩解）
- 手機號碼更換（Email 綁定備援；客服）
- 既有會員遷移（不影響既有 Email 會員）
- OTP 簡訊發送延遲（bacchusTask 排程）
- Email OTP 發送延遲/失敗（前端提示可改用密碼）
- 裝置遺失/汰換（重新啟用）
- 生物辨識安全性（OS 層安全；密碼變更 token 失效）
- 多裝置管理（後端記錄各裝置 token）
